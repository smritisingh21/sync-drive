# Analyzing HTTP/HTTPS Traffic with Wireshark

## Capture Steps
1. Start capture on the active NIC.
2. (Optional) Clear DNS cache: `ipconfig /flushdns`.
3. Browse the target site(s): test both **HTTP** and **HTTPS** pages.

## Display Filters
- HTTP requests: `http.request`
- By hostname (HTTP): `http.host == "example.com"`
- TLS/HTTPS handshake: `tls`
- SNI: `tls.handshake.extensions_server_name == "example.com"`
- Specific TCP conversation: Right-click a packet → **Follow** → **TCP Stream**

## What to Record
### For HTTP
- Method (GET/POST), **Host** header, **Request-URI**, user-agent
- Response status (200/301/404), `content-type`, `server` header

### For HTTPS
- TLS **ClientHello**: version, ciphers, **SNI**
- TLS **ServerHello**: selected cipher, ALPN (e.g., h2, http/1.1)
- Certificate: CN/SANs, validity
- Note that payload content is encrypted

## Performance/Health Clues
- TCP 3-way handshake RTT, TLS handshake time
- `tcp.analysis.retransmission` for loss
- `http.response.code >= 400` for errors
